## 一、基本数据类型

|数据类型|描述|内存大小|取值范围|
| :---:| :---: | :---: | :---: |
|char|字符数据类型|1 byte|-128 到 127 或 0 到 255|
|int|整型数据类型|2 or 4 byte|-32,768 到 32,767 或 -2,147,483,648 到 2,147,483,647（取决于系统|
|float|单精度浮点数|4 byte|1.2E-38 到 3.4E+38|
|double	|双精度浮点数	|8 byte	|2.3E-308 到 1.7E+308|
|_Bool	|布尔型	|1 byte	|0(假) 或  1(真)|
        注：
            i. 在科学计数法中，为了使公式简便，可以用带 “E” 的格式表示。
                例如 1.03乘10的8次方，可简写为“1.03E+08” 的形式。负号则代表乘10的负几次方；
            ii. 整型是一种数据类型，用于存储整数，即不包含小数部分的数值；
            iii. 单精度和双精度的主要区别在于它们的存储大小、精度和表示范围；
            iv. 8个bit组成1个byte，用于表示8位二进制数，即256种不同状态。

## 三、char

###### 1.主要作用和特点：

    (1) 存储单个字符：
            • char 类型通常用于存储单个字符，比如字母、数字、符号等。
            例如：char letter = 'A'; 这里的 letter 存储字符 'A'。
            
    (2) 存储 ASCII 值：
            • 在 C 语言中，字符实际上是存储为 ASCII 码 值的，
            ASCII 是字符和数值之间的对应表。
            例如,字符 'A' 对应的 ASCII 值是 65。
            char c = 'A'; 实际上 c 存储的是 65，而不是 'A' 本身。
        
        注：char letter = 'A'，letter存储的也是65。
            但当你使用 %c （格式化输出符号，用于表示(符)时，
            C 语言会将其解释为字符 'A'，而不是数字 65。

                char letter = 'A';
                printf("%c\n", letter);    // 输出 'A'
                printf("%d\n", letter);    // 输出 65    
       
            在这个例子中，当使用 %c 时，程序会把 letter 解释为字符并输出 'A'；
            而使用 %d 时，会把 letter 作为整数解释并输出它的 ASCII 值 65。

Q：为什么ASCII代码没有 1-31 ？

A：在 ASCII 码表中，代码 1-31 实际上是存在的，但它属于 控制字符（Control Characte(s)，而不是可打印字符。控制字符通常用于控制文本的显示或传输，而不是直接显示在屏幕上。
